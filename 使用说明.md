# 使用说明

本文档说明「糖尿病智能助手」平台的访问方式、测试账号、用户端与管理端操作流程及常见问题。

> 安装部署与开发调试见 [README.md](./README.md)、[开发指南.md](./开发指南.md)。

---

## 1. 平台简介

**糖尿病智能助手**为糖尿病患者及关注人群提供：

- 糖尿病科普问答与资讯阅读
- 健康档案管理与 AI 风险评估
- AI 模拟医生在线咨询
- 个性化饮食 / 运动 / 作息方案
- 四类生活打卡（食物、用药、运动、血糖）与 AI 行为分析
- 健康趋势分析与主动预警消息

> **免责声明：** 所有 AI 生成内容仅供参考，不能替代专业医生诊断与治疗建议。如有健康问题，请及时就医。

## 4. 用户端使用

用户端底部导航：**首页 · 生活打卡 · 资讯 · AI 助手 · 我的**。

### 4.1 注册与登录

1. 打开用户端，点击「登录 / 注册」
2. **注册**：填写手机号 → 获取邮箱验证码（需管理员配置邮件服务）→ 设置密码
3. **登录**：输入用户名与密码；登录后可使用风险评估、打卡、问诊等需认证功能

也可直接使用测试账号 `testuser` / `123456` 登录体验。

### 4.2 首页与科普

| 功能 | 路径 | 说明 |
|------|------|------|
| 首页 | `/home` | 轮播、快捷入口、热门科普视频 |
| 科普视频 | `/science-videos` | 视频列表与播放 |
| 健康资讯 | `/health-info` | 分类浏览、搜索、个性化推荐 |
| 资讯详情 | `/health-info/:id` | 正文阅读、收藏、**听全文** |
| AI 助手 | `/assistant` | 科普问答（流式回答）、快捷问题、**语音输入** |

以上页面除需收藏等操作外，多数可匿名浏览。

### 4.3 健康评估与方案

1. **个人中心 → 健康档案**：填写身高、体重、血糖、血压、家族史等
2. **风险评估**（`/health-evaluation`）：完成问卷并提交，等待 AI 生成报告（约 30~120 秒）
3. 报告完成后，消息中心会收到通知
4. **健康方案**（`/living-plans`）：基于档案与风险一键生成饮食 / 运动 / 作息方案，支持收藏与导出

### 4.4 生活打卡

1. 进入 **生活打卡**（`/checkin-records`），查看今日统计与连续天数
2. 选择打卡类型：
   - **食物**：手动录入、预设食物或 **拍照 AI 识别**
   - **用药** / **运动** / **血糖**：按表单填写
3. **打卡分析**（`/checkin-analysis`）：查看周期趋势与 AI 行为建议
4. **打卡提醒**（`/checkin-reminder-settings`）：配置四类提醒时段

### 4.5 AI 助手与问诊

1. **AI 助手**：输入健康问题或点击快捷问题；支持麦克风语音输入（需配置 DashScope）
2. **医师咨询**（`/consultation`）：选择 AI 医生 → 进入对话 → 描述症状
3. 所有 AI 回答页面均展示免责声明，不构成医疗诊断

### 4.6 个人中心与消息

| 功能 | 说明 |
|------|------|
| 个人中心 | 资料编辑、头像、健康档案、AI 月度趋势、异常预警 |
| 消息中心（顶栏铃铛） | 评估完成、方案生成、问诊回复、打卡分析、健康预警等通知 |
| 账户安全 | 修改密码、隐私设置、数据导出 |

## 5. 管理端使用

管理端地址：http://localhost:81/（或本地开发 http://localhost:5174）。

### 5.1 登录

使用管理员账号 **`admin` / `123456`** 登录。普通用户账号无法进入管理后台。

### 5.2 功能模块

| 模块 | 路径 | 功能 |
|------|------|------|
| 管理首页 | `/home` | 注册量、活跃用户、打卡量、问诊量等概览 |
| 数据统计 | `/statistics` | 注册/打卡趋势、用户画像与分布图表 |
| 资讯管理 | `/articles` | 资讯 CRUD、审核流转、**AI 生成初稿** |
| 科普视频 | `/videos` | 上传视频与封面、同步用户端首页 |
| 审计日志 | `/audit-logs` | 操作记录查询与 CSV 导出 |

### 5.3 资讯发布流程

1. 进入 **资讯管理**
2. 点击「AI 生成初稿」，输入主题与关键词，等待 Dify 流式生成 Markdown
3. 在编辑器中修改完善
4. 保存为草稿 → 提交审核 → 审核通过后发布
5. 已发布资讯自动出现在用户端资讯列表

### 5.4 视频发布流程

1. 进入 **科普视频管理**
2. 上传视频文件与封面图（系统自动解析时长）
3. 发布后同步至用户端首页「热门科普视频」

## 7. 业务闭环示意

```text
浏览科普 / 资讯
    ↓
填写健康档案 → AI 风险评估
    ↓
AI 问诊（可选）→ 生成健康方案
    ↓
日常打卡（食物/用药/运动/血糖）
    ↓
打卡 AI 分析 + 健康趋势预警
    ↓
个性化资讯推荐 + 主动干预消息
```

## 8. 常见问题

| 问题 | 可能原因与处理 |
|------|----------------|
| AI 功能无响应或超时 | 检查 Dify 是否运行；联系管理员核对 API Key 配置 |
| 科普回答质量差 | 知识库（Milvus）可能未导入；可换用更具体的问题 |
| 语音识别失败 | DashScope 未配置或音频无法公网访问 |
| 食物拍照识别失败 | 图片 URL 需公网可达（本地开发需 ngrok 等穿透） |
| 注册收不到验证码 | 邮件 SMTP 未配置；可改用测试账号登录 |
| 页面显示假数据 | 前端可能处于 Mock 模式，需关闭 `VITE_USE_MOCK` |
| 风险评估一直 loading | Dify 工作流未发布或 API Key 错误 |
| 管理端无法登录 | 须使用 `admin` 账号，非普通用户账号 |

技术排查细节见 [开发指南.md §9](./开发指南.md#9-常见问题开发)。
